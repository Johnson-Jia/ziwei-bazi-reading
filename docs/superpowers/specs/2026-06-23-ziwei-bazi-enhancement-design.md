## 1. 背景与目标

`ziwei-bazi-reading` 是一个交互式批命 Skill，紫微批命靠 LLM 读 `methods/ziwei-method.md` 方法论**灵活推演**。参考项目 `ziwei-samples-toolkit` 则是一个倪海厦体系的离线样本生成工具包，其 `lib/ziwei/db-analysis.ts`（368KB）、`patterns.ts`、`heming-knowledge.ts`、`corpus/verified/` 蕴含了高度系统的批命知识结构与批法骨架。

**核心洞察**：两个项目是互补关系，不是替代关系。
- `ziwei-samples-toolkit` 的强项 = 系统的知识矩阵、算法化格局识别、子午流注、合盘五步法、语料溯源分级。
- `ziwei-bazi-reading` 的灵魂 = LLM 灵活推演、双体系独立、强约束防幻觉、排盘引擎齐全。

**目标**：借鉴 samples-toolkit 的**知识结构与批法骨架**，增强本项目紫微批命的系统性与深度，同时体检补强八字方法论的真空缺。**借结构，不借断语原文**——所有内容重写为通用白话 + 派别/来源标注，不直接搬运断语库、不锁死倪师单体系。

## 2. 决策记录

| 决策点 | 选择 | 理由 |
|---|---|---|
| 体系口径立场 | **通用为主 + 派别标注** | 保留"通用+双体系独立"灵魂（SKILL 铁律第6条）；倪师专属断法以标注引入，用户可按派别取舍，覆盖最广 |
| 合盘功能归置 | **独立专题，按需加载** | 不污染单盘推演流程；SKILL 指向但仅按需 Read |
| 实施范围 | **紫微全做（6 项）+ 八字同步体检** | 全面增强 |
| 合盘文件组织 | **方案 B-1：单一 `heming-method.md` 双体系合盘专题** | 紫微部分 + 八字部分各自独立、互不引用术语；一次按需加载服务两种合盘需求 |
| 八字疾病口径 | **五行对应脏腑，独立术语** | 八字疾病只用木肝/火心/土脾/金肺/水肾逻辑，**不混入紫微的地支宫→脏腑逻辑**，保持双体系独立 |

## 3. 总策略：统一标注体系

贯穿所有新增内容。每条带术数来源的断语必须能溯源：

| 标注 | 含义 | 用法 |
|---|---|---|
| `[古诀]` | 《紫微斗数全书》等古籍通识口诀（跨派共识） | 引用传统口诀时 |
| `[命理共识]` | 各派通用的方法论概念（非某派独创） | 描述通用技法时 |
| `[倪海厦视角]` | 倪海厦（1953-2012）专属断法 / 天纪出处 | **明确标为某派视角，不冒充通识，不编造原话** |

**纪律**：不编造"倪海厦说 XXX"这种冒名原话。若需引用倪师内容，用 `[倪海厦视角]` 标注并尽量注明出处（如"天纪05案例"），表述为派别观点而非绝对真理。

| 状态 | 批法倾向 |
|---|---|

**第二部分：八字合婚**（独立术语，不提宫位/星曜/四化）

3. **五行互补**：双方日主五行及命局五行是否互补（一方所缺为另一方所旺）`[命理共识]`

## 5. 设计原则与约束

1. **双体系独立铁律不破**：`heming-method.md` 内紫微部分与八字部分各自独立、互不引用术语；紫微文档不提十神/大运/喜用神，八字文档不提宫位/星曜/四化。

6. **不动排盘引擎与铁律架构**：本次仅增强 `methods/`、新增 `heming-method.md`、微调 `SKILL.md`。不修改 `scripts/`、`data/`、`templates/`。

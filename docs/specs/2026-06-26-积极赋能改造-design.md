# 积极赋能改造设计 · ziwei-bazi-reading 技能

> 日期:2026-06-26 ｜ 状态:设计待审 ｜ 方案:中等·机制化 ｜ 哲学:顺势而为此乃大道

## 一、背景与目标

**现状矛盾**:技能方法论层早已规定"正向引导"——SKILL.md 铁律第 8 条「趋吉避凶·正向引导·每个凶配一条出路」、`ziwei-method.md` 第十一节「化忌转课题/煞星转能量/凶格配出路」、`bazi-method.md` 第十五节、corpus 倪师「以果决行」「命+地+人 > 天命」。**但生成层(gen_*.js 的 INTERP 表、格局卡、LLM 解读)完全没机制化落实**——产出仍是「财凶:破财/投资损/感情波折」「灾凶:血光/意外」式的纯断凶,凶无出路、无行动指引,易制造焦虑与宿命感。

**目标**:在**保证推演正确(排盘/吉凶判断零变化)**的前提下,把正向引导方法论**机制化**——新增「积极赋能语料层」,让每个关键节点(格局/宫位四化/喜忌特质/大运大限/流年/合盘)的吉凶判断自动配「如实断 + 转化路径 + 行动指引 + 心态」,产出"知己·顺势·行动"的积极命书。

**定位**:命理不作宿命预测,作**自我认知 + 顺势参考**,引导人扬长避短、积极行动。

## 二、核心原则(不可违背)

1. **推演层零改动**:排盘引擎(iztro / tyme4ts / bazi_core)、推演方法论(methods/)、吉凶判断逻辑——一行不改,守住"正确"。
2. **双轨表达**:每条命理产出 = `judgment`(如实断,不粉饰)+ `transform / action / mindset`(赋能)。保准确性,又落地积极引导。
3. **禁绝对化/凶吓**:禁「必凶/死局/家破/必破」,用「需注意/宜 X/课题在 X」。
4. **每凶配出路**:无裸断凶,每个凶象必配转化 + 行动。

## 三、架构(数据流)

```
改前: 排盘 → 推演(格局/喜忌/运势) → 吉凶判断 → [直接输出]            → 命书(断凶倾向)
改后: 排盘 → 推演(格局/喜忌/运势) → 吉凶判断 → 【积极赋能层·新增】 → 命书(双轨:如实断+积极引导)
                                                    ↑ 查 data/empower.json
```

**关键**:推演与判断不动,只在「判断 → 产出」之间插一层赋能映射。

## 四、积极赋能语料层(`data/empower.json`)

结构化映射库,按节点分 6 类:

```json
{
  "geju":         { "机梁格": {...}, "武贪格": {...}, "半空折翅": {...} },
  "palace_sihua": { "财帛化忌": {...}, "父母煞忌": {...} },
  "bazi_trait":   { "身弱财多": {...}, "印透无根": {...} },
  "dayun":        { "财官忌运": {...}, "食伤喜运": {...} },
  "liunian":      { "财官忌年": {...}, "食伤喜年": {...} },
  "heming":       { "日柱冲克": {...}, "喜用互补": {...} }
}
```

每条统一结构:
```json
{
  "judgment": "如实断(保留命理判断,不粉饰)",
  "transform": "转化路径(凶→功课/吉→窗口)",
  "action": ["具体行动1", "具体行动2"],
  "mindset": "心态(如:被动破财非自责)"
}
```

**来源**:80% 从 `ziwei-method.md` 第十一节"紫微凶象转化速查"、`bazi-method.md` 第十五节、corpus 凶象转化**直接抽取**(已有文本→结构化);20% 补**通用转化模板**(查不到的象兜底)。

**示例**:
```json
"财帛化忌": {
  "judgment": "正财受阻、破财风险",
  "transform": "财路转向技术生财,忌投机取巧",
  "action": ["稳健理财", "技术深耕变现", "远离合伙/担保/高风险投资"],
  "mindset": "财多压身多因被动卷入,非个人贪财,勿自责"
}
```

## 五、4 个接入点(赋能层如何被用上)

1. **INTERP 解读表升级**(gen_*.js,最高频):现有 `INTERP={'财凶':'断语'}` 纯断 → 每条维度凶查 `empower.json` 配 `transform+action`。
2. **格局卡配出路**(gen_ziwei_full 凶格卡):凶格(type:caution/bad)自动附对应象的转化。
3. **命书新增「积极引导」专区**(gen_*_full):三篇,由推演结果 + empower 自动生成(详见第六节)。
4. **LLM 解读 prompt 框架**(解读 JSON 生成时):固化四铁律——双轨输出 / 积极基调 / 禁凶吓 / 引 empower.json。

## 六、命书「积极引导」专区结构

命书末尾(综合建议前)加一大块,分三篇:

| 篇 | 内容来源 | 产出 |
|---|---|---|
| **知己·禀赋扬长** | 命/身/官禄主星 + 八字日主喜忌 | 禀赋定位 + 发力方向 + 滋养源(学习/合作/技术创造) |
| **顺势·节奏功课** | 大运基调 + 流年喜忌 | 发力期(顺势)/ 守成期(功课)逐年指引 |
| **行动·避坑指南** | 喜忌 + caiYinRelation | 扬长 + 避坑(远离合伙/投机等破财模式) + 心态 |

**前后对比**:
- 改前:「2028戊申:财官忌神,综合 -3(凶)」← 止于断,剩焦虑。
- 改后:「2028戊申:财官重(如实);**转化**:守成功课年,保本蓄力;**行动**:不投资/不担保/还债为主/技术深耕;**心态**:守得住即赢,为中年翻身打地基」← 同判断,积极行动框架。

## 七、全节点覆盖

| 产出脚本 | 接入赋能层的关键节点 |
|---|---|
| 完整命书 gen_ziwei/bazi_full | 格局卡、六维每条、大限基调、流年表、+「知己/顺势/行动」专区 |
| 流年 gen_*_liunian | 每年太岁十神/宫象 → 配 transform+action;折线图凶年低谷点带"功课"注解 |
| 合盘 gen_heming_* | 双方关键象(刑克/互补/喜用互助)→ 配转化(刑克→相处功课、互补→协同发力) |

## 八、分阶段实施

**Phase 1 · 核心底座**(先跑通,最有价值)
- 建 `data/empower.json`(6 类约 30-50 条,从 methods/corpus 抽)
- 完整命书接入:INTERP 升级 + 格局卡配出路 + 「知己/顺势/行动」专区
- 通用转化模板(兜底)

**Phase 2 · 流年**
- 流年表/折线图注解接入赋能层(每年配转化)
- 流年解读 prompt 固化

**Phase 3 · 合盘 + 机制固化**
- 合盘关键象接入
- 解读 JSON 生成的 prompt 框架沉淀(四铁律)
- `SKILL.md` 铁律第 8 条强化为"机制化(查 empower.json)"——把"要求"变"机制"

## 九、验收标准(每 Phase 通用)

1. **推演输出零变化**:排盘/吉凶判断与改造前字节级一致(回归对比)。
2. **每条凶/特质/运势节点配 transform+action**(无裸断凶)。
3. **通篇零绝对化/凶吓**表述(无"必凶/死局/家破/必破")。
4. 双轨:judgment 仍在(准确性不丢)+ 赋能附后。

## 十、不做什么(边界)

- 不改推演引擎/方法论/吉凶判断逻辑。
- 不去掉吉凶判断(双轨,非粉饰太平)。
- 不做"重度重定位"(吉凶降级/技能重定义)——保持命理本位 + 积极引导叠加。
- 不引入新依赖(empower.json 纯数据,gen 改为读 JSON)。

## 十一、风险与对策

- **empower.json 覆盖不全** → 通用转化模板兜底(查不到的象用"此为 X 课题,宜 Y"通用句式)。
- **接入 gen_*.js 引入回归** → 每 Phase 用命主甲戌盘做字节级回归对比(推演输出必须一致)。
- **双轨可能显得啰嗦** → 命书呈现上 judgment 用小字/灰色,transform+action 主色,主次分明。

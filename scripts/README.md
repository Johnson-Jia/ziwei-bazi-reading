# 排盘脚本（scripts/）

「生辰 → 命盘数据」的排盘脚本，紫微斗数与八字**均已实现**，是全流程的数据入口。

## 数据流（双引擎，均已打通）

```
公历生辰 + 时辰/时分 + 性别
        │
   ┌────┴─────┐
   ▼          ▼
 paipan_ziwei.js        bazi.js
 紫微(iztro v2.5.8)     八字(tyme4ts v1.5.2 + 关系层 + 神煞层)
   │          │
   └────┬─────┘
        ▼
 结构化命盘 JSON  →  methods/* 推演  →  templates/* 渲染
        ▲
        │ 读取
 data/*.json（藏干/四化/节气静态表，与脚本解耦）
```

## ✅ 紫微：paipan_ziwei.js（iztro）

```bash
node scripts/paipan_ziwei.js                            # 演示命主 1993-10-20 戌时 男
node scripts/paipan_ziwei.js 2000-8-16 2 女 true zh-CN  # 任意(参数: 日期 时辰0-12 性别 闰月 语言)
```
- 时辰序号：0早子…10戌…12晚子
- 输出：四柱/五行局/命主身主/十二宫(主星亮度四化/辅星小星/宫干/大限/小限/长生)

## ✅ 八字：bazi.js（tyme4ts + 关系层 + 神煞层）

```bash
node scripts/bazi.js                          # 演示命主 1993-10-20 19:10 男
node scripts/bazi.js 1985 10 15 14 30 男      # 任意(参数: 年 月 日 时 分 性别; tyme4ts按精确时分自动定时辰)
```
- 三层架构：**tyme4ts**（四柱/十神/藏干/地势/胎元命宫身宫/精确节气起运/大运）+ **bazi_relations**（三合/六合/冲/刑/害/破/天干合冲）+ **shensha**（15 命局神煞）
- 输出：四柱/日主/各柱十神藏干地势/胎元命宫身宫/起运(实年+交运日)/八步大运(虚岁)/地支天干关系/命局神煞
- 起运虚岁换算见 `data/jieqi.json`；childLimit 字段给实年，daYun.startAge 为虚岁

## ✅ 八字流年逐年断：bazi_liunian.js

```bash
node scripts/bazi_liunian.js                                  # 演示命主 流年2012-2035
node scripts/bazi_liunian.js 1985 10 15 14 30 男 2000 2030    # 任意生辰 + 年份范围
```
- tyme4ts 取**流年太岁干支 / 虚岁 / 所在大运 / 流年支与命局关系**；自研层做 **旺衰判定 → 喜用忌神 → 流年干支十神 → 吉凶**
- `verdict`：喜用年倾向吉、忌神年倾向凶、参半为平（概率性，非定论）
- **已验证**：命主 2018戊戌→凶（忌财，应父亲手术）、2022壬寅→吉（喜印比，应得学位），与人工命理分析一致；旺衰判身弱与定性吻合
- ⚠ 旺衰用简化三得模型；从格/调候/特殊格局命局须人工复核（见 `methods/bazi-method.md`）

## 交互式运势报告（布局优化）

两套体系均采用**左侧大运/大限列表（点击）+ 右侧流年十维度+建议**的交互布局：

**八字**（`gen_liunian_html.js` → `八字运势-YYYY.html`）：
- 左侧：大运列表卡（干支+年龄+十维度基调小字，当前大运红框标"当前"）
- 点击大运 → 右侧展开该大运的流年列表
- 每流年卡：太岁+虚岁+整体verdict+**十维度badge**（妻财子禄父身友考宅灾·吉凶色块）+**针对性建议**（凶维度→防范、吉维度→顺势）

**紫微**（`gen_ziwei_liunian_html.js` → `紫微运势-YYYY.html`，按紫微特点）：
- 左侧：大限列表卡（干支+**大限宫位主星**+年龄，紫微特有）
- 点击大限 → 右侧流年列表
- 每流年卡：**流年命宫地支+流年四化（禄权科忌）**+十维度宫象（hover看该宫星象）+建议
- 紫微特点：流年命宫流转、流年四化、宫位星象（原局星+流耀+四化入宫）

两报告默认定位到命主当前大运/大限，手机端自适应（大运列表横向滚动）。

## 十维度运势（妻财子禄父身友考宅灾）— 双体系

每个大运/流年输出**十领域吉凶**，紫微+八字双体系交叉：

- **紫微流年十维度**（`ziwei_liunian.js`，iztro horoscope 运限盘）：流年十二宫位映射十维度（妻=流年夫妻宫/财=流年财帛宫/子=流年子女宫…），每宫看原局星吉煞庙旺+流年星+流年四化入宫。擅宫位星象（财/子）。
- **八字流年十维度**（`bazi_core.js` baziDims）：十神喜忌映射（妻/财/父=财星、子/禄/灾=官杀、考=印、身=日主受流年生克、友=比劫）。擅十神喜忌（父/灾/考）。
- **大运十维度**：大运干支十神对各维度基调（daYun.dims）。

**交叉验证**（命主真实应事）：
- 2018 父亲腿伤：紫微父·平（流年宫位局限）＋**八字父·凶✓**（偏财忌）→ 八字补紫微
- 2018 副业烧钱：紫微财·凶✓ ＋ 八字财·凶✓ → 双体系一致
- 2020 胎停：紫微子·凶✓（破军陷地劫）＋八字子·凶✓（官杀=子）→ 双体系一致
- 2022 得学位：紫微考·平 ＋ **八字考·吉✓**（印喜）→ 八字更准

**HTML**：`gen_liunian_html.js` 流年表后附「十维度矩阵」（年×十维度，吉凶颜色编码）。

## 模块结构（DRY）

- `bazi_core.js` — **八字核心引擎**（排盘+旺衰+喜用+流年逐年断），被 `bazi_liunian.js`(→JSON) 与 `gen_liunian_html.js`(→HTML) 共用，逻辑单点维护

## 引擎与 vendor

两引擎均已 **vendor 内置**于 `scripts/vendor/`（离线可用，无需 install/build）：

| 引擎 | 体积 | 依赖 | vendor 路径 |
|---|---|---|---|
| iztro v2.5.8 | ~5.2MB | dayjs/i18next/lunar-lite/lunar-typescript(已含) | `vendor/iztro/` + `vendor/node_modules/` |
| tyme4ts v1.5.2 | ~352KB | 无(unbuild bundle) | `vendor/tyme4ts/dist/lib/index.cjs` |
| 自研关系/神煞层 | ~16KB | 无 | `vendor/bazi/bazi_relations.js`, `shensha.js` |

详情、升级方法、许可证见 `vendor/README.md`。换环境可设 `IZTRO_LIB` / `TYME4TS_LIB` 指向其他副本。

## 设计约定

- **数据与脚本解耦**：静态表（藏干/四化/节气）放 `data/*.json`，脚本读取不硬编码
- **输出统一 JSON**：字段供 `methods/` 读取、`templates/` 渲染
- **引擎可替换**：路径用 `__dirname` 默认 vendor，环境变量可覆盖
- **两套独立**：紫微脚本只 require iztro，八字脚本只 require tyme4ts+自研层，互不引用
